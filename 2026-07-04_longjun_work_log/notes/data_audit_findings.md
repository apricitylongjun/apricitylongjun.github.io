# 数据审计补充发现 — features_all.npz 特征质量核实

> 本文件**补充** `notes/data_audit.md`，**不覆盖**它。
> 已有 data_audit.md 选定 `features_all.npz` 为数据源；本核实发现其存在特征质量缺陷，供数据源决策参考。
- 核实日期：2026-07-04
- 方法：`np.array_equal` 逐切片精确比对（非统计推断）

## 核实结论（确凿）
`features_all.npz` 的 data `(4030, 4, 256)` 的 4 个切片：

| 切片 | 状态 | 证据 |
|---|---|---|
| [0] | 有效（连续特征） | 非零占比 1.0，唯一值 ~101 万，mean 0.062 |
| [1] | **与 [0] 完全相同** | `np.array_equal(切片0, 切片1)=True`，最大绝对差 = 0.0 |
| [2] | **全为 0** | `np.all(切片2==0)=True`，唯一值数 = 1 |
| [3] | 有效（离散特征） | 唯一值仅 24 个，非零占比 0.178，范围 0–25 |

→ **实际有效切片 = 2 / 4**（[0] 与 [3]；[1] 冗余、[2] 空）

## 与已有 data_audit.md 的分歧
- 已有报告：选 features_all，理由"(4,256) 可直接喂 1D-CNN、34 类均衡"——**正确，但未做 per-slice 检查**
- 本核实：features_all 有 50% 切片损坏/冗余 → 实际信息量低于"4 通道"的名义维度
- 已有报告其余分析（被试/session 结构、R1 标签 1..34→0..33 remap、R3 归一化只用 train）**均正确，保留**

## 对预实验的影响
- features_all **仍可用**（切片 0+3 有信息，34 类分类能跑通流程），但相当于只用 ~2 个有效特征通道，信号偏弱
- 若要更强信号/更严谨，可换 `preprocess_features/*_dynamic_data.npy (N,4096)`：4096 维全有效、39 session / 7867 样本
